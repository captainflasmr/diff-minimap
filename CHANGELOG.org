This document contains the release notes for diff-minimap.

* Versions

** 0.8.0 — <2026-05-30>

Interactive mouse navigation from the minimap.

- Drag with =mouse-1= within the minimap to scrub the source buffer: the grabbed row scrolls the original buffer immediately and the view tracks the pointer continuously until release (=diff-minimap--drag-handler=).
- Scroll the source buffer with the mouse wheel / two-finger trackpad scroll over the minimap.  Add =diff-minimap-wheel-scroll-lines= (default 3) to control the step.
- Decode the clicked row from the minimap buffer explicitly (=diff-minimap--event-source=) rather than the current buffer, fixing click-to-jump and right-click hunk preview, which silently did nothing because mouse commands run with the source buffer current.  Clicks landing on a row's trailing newline now fall back to the row start.
- Fix a misplaced closing paren that folded =diff-minimap--post-command= and the entire mouse-handler / hunk-preview block into a single form, so those commands never loaded.

** 0.7.1 — <2026-05-17>

Refined hunk preview and toggle-all previews command.

- Add =diff-minimap-show-all-hunks= (bound to =C-c P=) to toggle all diff previews in the current buffer at once.
- Refine =diff-minimap-show-hunk= to strictly exclude context lines (using =--unified=0=) and replace original buffer text visually (using the =display= property).
- Make hunk previews persistent: previews (both individual and batch) now remain visible until manually toggled off, providing a stable view during edits.
- Improve sub-hunk extraction to correctly track line numbers for pure deletion blocks, ensuring the preview appears when point is on the line of deletion.

** 0.7.0 — <2026-05-17>

Dual-column Ediff minimap and granular overlay scanning.

- Add automatic dual-column minimap for Ediff sessions.  When Ediff is active, the minimap displays Buffer A in the left column and Buffer B in the right column, vertically aligned.
- Capture granular "fine diffs" (word-level changes) directly from Ediff's buffer highlights.
- Thematic colour synchronization: minimap colors derive from Ediff's own faces (=ediff-current-diff-A=, =ediff-fine-diff-B=, etc.), ensuring the minimap exactly matches the buffer's visual state.
- Seamless Ediff integration: automatic minimap startup via =ediff-startup-hook= and teardown via =ediff-quit-hook=.
- High-fidelity viewport sync: the minimap viewport follows Ediff navigation (=n= / =p=) from the control panel.
- Virtual coordinate mapping: ensure Buffer A and Buffer B remain horizontally synchronized in the minimap representation even when hunks contain differing numbers of physical lines.

** 0.6.0 — <2026-05-15>


Inline overlay hunk preview with per-change narrowing.

- Replace popup side-window in =diff-minimap-show-hunk= with an inline overlay that displays the fontified unified diff hunk as virtual text (via =before-string=) directly in the source buffer above the hunk region.  The hunk lines are highlighted with =diff-minimap-hunk-region= face.
- Narrow the displayed diff to only the specific change group (contiguous =-= /=+= block) that contains point, ignoring unrelated changes and context lines in the same git hunk.  Pure removals (no =+= lines) fall back to the full hunk.
- Toggle the preview on/off by calling =diff-minimap-show-hunk= again.
- Auto-remove the overlay when point moves outside the hunk range.
- Changes are backward-compatible to the caller: =C-c p=, =diff-minimap-mode-map= binding, and right-click on the minimap all work identically.

** 0.5.0 — <2026-05-14>

Optional diff-hl, VC backend, built-in fringe indicators.

- Remove hard dependency on =diff-hl= — the minimap now runs without it.  Add =diff-minimap-diff-backend= (=auto= | =diff-hl= | =vc=) to choose the diff-data source.  =auto= tries =diff-hl= first then falls back to git.  =vc= uses git directly (no =diff-hl= required).
- Add =diff-minimap-fringe-indicators= (t / nil) — when enabled, the package paints coloured fringe bitmaps in the source buffer on every changed line, matching the minimap colours.  Works with any backend.
- Add =diff-minimap-fringe-side= (=left= | =right=) to choose which fringe holds the indicators.
- Fix =diff-minimap--cache-faces= crash when =diff-hl= faces are undefined (guard =face-attribute= calls with =facep=).
- Add =diff-minimap-next-hunk= and =diff-minimap-previous-hunk= for navigating between diff hunks (pulses the full hunk range on arrival).  Bound to =C-x v ]= and =C-x v [= (vc-prefix-map) and =M-]= / =M-[= (minor-mode map).
- Add =diff-minimap-search-highlights= (t / nil) — when enabled, isearch matches appear as yellow bars in the minimap, updated live on every keystroke.
- Add =diff-minimap-show-hunk= command to preview the diff hunk at point in a popup window (=C-c p= in the source buffer, =mouse-3= on the minimap).
- When =diff-minimap-mode= is active, ~vc-diff~ (=C-x v =) automatically positions point at the hunk corresponding to the source line.
- Fix =diff-minimap--parse-git-diff= merge-ranges bug with adjacent hunks of the same type (input order was reversed before merging).

** 0.4.0 — <2026-05-10>

Overlay priority layering, filled default, demap integration, and removal of outline.

- Replace text-property-based diff colouring with per-row overlays (priority 3) so diffs sit between the viewport fill (priority 1) and cursor (priority 10).
- Change =filled= viewport style priority from 5 to 1 — diffs now show through the fill instead of being obscured.  Make =filled= the new default.
- Change =stipple= viewport style priority from 3 to 2 — diffs show through the stipple bitmap, matching the described intent ("diffs visible through").
- Remove =outline= viewport style (box outline around the viewport).
- Add =diff-minimap-toggle-with-demap= command that opens/closes the diff minimap and =demap= together for a VSCode-like two-window layout.

** 0.3.0 — <2026-05-09>

Global mode, multi-buffer support, and rendering accuracy.

- Change =diff-minimap-mode= from buffer-local to global — once toggled on, the minimap follows the selected buffer and updates automatically on switch. No need to toggle per file.
- Handle Dired and other buffers without =diff-hl-mode=: the minimap content is cleared on entry, re-rendered on return to a file buffer.
- Protect minimap window from =delete-other-windows= (=C-x 1=) via explicit =no-delete-other-windows= window parameter.
- Protect minimap window width from =split-window-right= (=C-x 3=) via =window-size-fixed= buffer-locally in the minimap buffer.
- Add =diff-minimap--buffer-lines= helper that compensates for the trailing newline Emacs adds on save, correcting the =total-line-count / nrows= scale so the last content line (and its diff hunk) maps to the last minimap row.
- Align =render=, =update-viewport=, and =post-command= to all use =window-text-height= for minimap row counts — previously =render= used =window-height= (includes the mode line), inflating =nrows= and misaligning the viewport overlay range with the rendered content.
- Change =stipple= viewport style edge bars from solid =:background= to =:overline= / =:underline= in the default foreground — the bottom edge bar no longer covers the last row's diff colour underneath.
- Add deferred re-render (0.2 s timer) after buffer switches to catch =diff-hl= overlay updates that arrive asynchronously when visiting a previously unseen file.
- =diff-minimap--update-viewport= now also handles non-=diff-hl= buffers by clearing overlays when entering them.

** 0.2.1 — <2026-05-09>

Customisation and reliability.

- Add =diff-minimap-colour-source= (diff-hl | diff) to choose whether diff highlights use =diff-hl-insert/change/delete= faces or the built-in =diff-added/changed/removed= faces as the primary colour source.
- Add =diff-minimap-stipple-pattern= with five predefined bitmap patterns (dots-sparse, diagonal, crosshatch, checkerboard, dots-dense) and a custom =(WIDTH HEIGHT DATA)= option.
- Fix viewport overlay not updating after =beginning-of-buffer= and similar commands: compute we-row from window-text-height instead of window-end (stale during post-command-hook), and fall back to centring the viewport around the cursor when window-start is stale.

** 0.2.0 — <2026-05-09>

Performance rewrite and viewport style options.

- Replace idle-timer scheduling (0.02 s) with direct =post-command-hook= for instant cursor/viewport reflection — no more perceptible lag.
- Split rendering into two tiers: full buffer rebuild only when diff content changes (edit / save / theme switch); viewport and cursor positions are updated via cheap overlay moves on every command.
- Cache face colours in defvars so the nine =face-attribute= calls happen once per full render instead of once per minimap row.
- Add =diff-minimap-viewport-style= defcustom (edge-bar | filled | outline | stipple) controlling how the viewport indicator is drawn.
- Theme-change detection: track =custom-enabled-themes= and trigger a full rebuild when the theme switches.
- Fix =diff-minimap--update-viewport= called from inside =with-current-buffer mm-buf= context, causing wrong buffer measurements on first toggle.

** 0.1.0 — <2026-05-08>

Initial release.

- VSCode-style minimap showing diff-hl hunks across the entire buffer.
- Colour-coded rows: added (green), changed (yellow), removed (red).
- Visible viewport highlight and current-line marker.
- Click-to-jump navigation from minimap to source buffer.
- Buffer-local minor mode keeps the minimap in sync on scroll and save.
- =diff-minimap-toggle= command bound to =C-x v m=.
- Customisable font scale, width, and side placement.

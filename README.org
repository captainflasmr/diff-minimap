#+title: diff-minimap — VSCode-style minimap for Emacs
#+author: James Dyer
#+email: captainflasmr@gmail.com
#+language: en
#+options: ':t toc:2 num:nil
#+startup: showall

A VSCode-style minimap for Emacs that colour-codes every line by its diff status (added, changed, removed) while also marking the visible viewport and current cursor row.  Can use =diff-hl= overlays or git directly as the diff-data source.  Optionally paints coloured fringe indicators in the source buffer.  Can be used standalone or alongside =demap= for a two-panel minimap layout.

* Installation

Drop =diff-minimap.el= onto =load-path= and require it:

#+begin_src emacs-lisp
(add-to-list 'load-path "~/.emacs.d/offline-packages/local-packages/diff-minimap")
(require 'diff-minimap)
#+end_src

Or with =use-package= (no =:ensure= since it is a local package):

#+begin_src emacs-lisp
(use-package diff-minimap
  :load-path "~/.emacs.d/offline-packages/local-packages/diff-minimap")
#+end_src

* Usage

~M-x diff-minimap-toggle~ or ~C-x v m~ opens the minimap in a side window.  The same command closes it.

The minor mode =diff-minimap-mode= is automatically activated when the minimap opens and deactivated when it closes.  It is a global mode — once toggled on, the minimap follows the selected buffer automatically, re-rendering for each file that has a diff backend available, and clearing when entering Dired or other buffers without changes.

** Mouse navigation

The minimap responds to the mouse so you can move around the source
buffer directly from it:

| Action                    | Effect                                                       |
|---------------------------+--------------------------------------------------------------|
| Click (=mouse-1=)         | Jump the source buffer to the clicked row                    |
| Drag (=mouse-1=)          | Scrub the source view continuously while the button is held  |
| Wheel / two-finger scroll | Scroll the source buffer (=diff-minimap-wheel-scroll-lines=) |
| Right-click (=mouse-3=)   | Jump to the row and show its diff hunk preview               |

** Hunk navigation

~diff-minimap-next-hunk~ and ~diff-minimap-previous-hunk~ jump between
diff hunks using the configured backend (the full hunk range pulses on
arrival).  Two sets of keybindings are available when the minimap is
open:

| Keybinding | Command                          |
|------------+----------------------------------|
| =C-x v ]=  | =diff-minimap-next-hunk=         |
| =C-x v [=  | =diff-minimap-previous-hunk=     |
| =M-]=      | =diff-minimap-next-hunk=         |
| =M-[=      | =diff-minimap-previous-hunk=     |
| =C-c p=    | =diff-minimap-show-hunk=         |

The =M-]= and =M-[= bindings are in =diff-minimap-mode-map= so they are
only active while the minimap minor mode is enabled.

** Inline diff preview

~diff-minimap-show-hunk= (=C-c p=) renders the `git diff` for the hunk
at point as an inline overlay directly in the source buffer, highlighted
via =diff-mode= font-lock.  The overlay appears above the hunk as virtual
text — it never modifies the underlying buffer.

Only the specific change group (the contiguous =-= /=+= block) containing
point is shown, without surrounding context or unrelated changes from the
same git hunk.  Call the command again to dismiss the overlay, or simply
move point outside the hunk range and it disappears automatically.

You can also right-click (=mouse-3=) on a minimap row to jump to the
source line and show its diff preview in one step.

** Ediff integration

When an Ediff session starts (=ediff-files=, =ediff-buffers=, etc.), the
minimap automatically transforms into a dual-column layout.  The visible
viewport follows your navigation (=n= / =p=) from the Ediff control
panel.

- **Dual columns**: Left column reflects Buffer A, right column reflects
  Buffer B.
- **Granular scanning**: The minimap captures word-level "fine diffs"
  directly from Ediff's buffer highlights.
- **Thematic colors**: Minimap colors are derived from Ediff's own faces
  (=ediff-current-diff-*=), ensuring they match your theme's Ediff
  highlights.

The minimap automatically hides when the Ediff session is closed.

* Demap integration

If you also use the =demap= minimap package, the ~diff-minimap-toggle-with-demap~ command opens demap on the right and diff-minimap on the left (set =diff-minimap-side= to ='left= beforehand) for a VSCode-like split: demap shows a zoomed-out code overview, diff-minimap shows the colour-coded diff ribbon.

#+begin_src emacs-lisp
(setq diff-minimap-side 'left)
;; then: M-x diff-minimap-toggle-with-demap
#+end_src

* Viewport styles

The viewport indicator can be selected via =diff-minimap-viewport-style=.  Available styles:

| Style      | Description                                                                  |
|------------+------------------------------------------------------------------------------|
| =edge-bar= | Solid bars at the top and bottom rows of the viewport.                       |
| =filled=   | Solid background across the entire viewport (diffs visible on top, default). |
| =stipple=  | Bitmap pattern fill + edge bars (diffs visible through).                     |

Set it in your config:

#+begin_src emacs-lisp
(setq diff-minimap-viewport-style 'stipple)
#+end_src

* Customisation

** User options

| Variable                         | Default      | Description                                                  |
|----------------------------------+--------------+--------------------------------------------------------------|
| =diff-minimap-font-scale=        | 0.1          | Fraction of default font height for the minimap (1.0 normal) |
| =diff-minimap-width=             | 6            | Width in characters of the side window (at the small font)   |
| =diff-minimap-side=              | 'right       | Side to display the minimap (=left= or =right=)              |
| =diff-minimap-viewport-style=    | 'filled      | How to render the viewport indicator (see above)             |
| =diff-minimap-stipple-pattern=   | 'dots-sparse | Bitmap pattern for the =stipple= viewport style              |
| =diff-minimap-diff-backend=      | 'auto        | Diff data source: =auto=, =diff-hl=, or =vc=                 |
| =diff-minimap-fringe-indicators= | t            | Show coloured fringe bitmaps on changed lines                |
| =diff-minimap-fringe-side=       | 'left        | Which fringe (=left= or =right=)                             |
| =diff-minimap-search-highlights= | t            | Show isearch matches as yellow bars in the minimap           |
| =diff-minimap-colour-source=     | 'diff-hl     | Prefer =diff-hl= fringe faces or built-in =diff= faces       |
| =diff-minimap-wheel-scroll-lines= | 3           | Source lines scrolled per wheel notch over the minimap       |

See the [[#stipple-patterns][stipple pattern table]] below for predefined values.

** Stipple patterns

The =stipple= viewport style uses an XBM bitmap as a textured background over the viewport.  Predefined patterns are available via =diff-minimap-stipple-pattern=:

| Value          | Description                   | Coverage |
|----------------+-------------------------------+----------|
| =dots-sparse=  | Diagonal single-pixel dots    | ~12 %    |
| =diagonal=     | Thin diagonal lines           | ~25 %    |
| =crosshatch=   | Alternating pixel cross-hatch | ~50 %    |
| =checkerboard= | Checkerboard                  | ~25 %    |
| =dots-dense=   | Denser dot pattern            | ~25 %    |

To use a custom stipple, set =diff-minimap-stipple-pattern= to a list of the form =(WIDTH HEIGHT DATA)= where DATA is an XBM bitmap string (bytes arranged left-to-right, top-to-bottom):

#+begin_src emacs-lisp
(setq diff-minimap-stipple-pattern
      '(8 8 . ,(apply #'string '(#x80 #x40 #x20 #x10 #x08 #x04 #x02 #x01))))
#+end_src

** Colour source

The =diff-minimap-colour-source= option controls which Emacs faces are queried for the diff highlight colours:

- =diff-hl= (default) queries =diff-hl-insert=, =diff-hl-change=, =diff-hl-delete= first, falling back to =diff-added=, =diff-changed=, =diff-removed=.
- =diff= reverses the priority: built-in diff faces first, diff-hl faces as fallback.

Both paths end with the same hardcoded hex fallbacks (=#006600=, =#666600=, =#660000=) so the minimap always has colour regardless of theme.

Other derived colours (=norm-bg=, =view-bg=, =cur-bg=) always come from =fringe=, =region=, and =cursor= respectively.

** Diff backends

The =diff-minimap-diff-backend= option controls where the minimap gets its diff data:

- =auto= (default) — tries =diff-hl= overlays first; if =diff-hl-mode= is not active, falls back to the git-based VC backend.
- =diff-hl= — reads =diff-hl= =diff-hl-hunk-type= overlay properties.  Requires =diff-hl= to be loaded and =diff-hl-mode= enabled.
- =vc= — runs =git diff --unified=0 HEAD= directly.  No =diff-hl= needed at all.  Works on any file tracked in a git repository.

The VC backend parses unified diff output into per-line change types (added / changed / removed) and supports both tracked files with uncommitted changes and staged files.

** Fringe indicators

When =diff-minimap-fringe-indicators= is non-nil (the default), the package paints coloured fringe bitmaps in the source buffer on every changed line, matching the minimap colours.  Three distinct bitmap shapes distinguish the diff types: full-width bar for additions, mid-width bar for changes, narrow bar for removals.

Set =diff-minimap-fringe-side= to ='left= (default) or ='right= to choose which fringe holds the indicators.

The fringe faces are updated whenever the minimap re-caches its colours (on theme switch, buffer switch, or explicit render), so they always stay in sync with the minimap palette.

To disable fringe indicators entirely:

#+begin_src emacs-lisp
(setq diff-minimap-fringe-indicators nil)
#+end_src

** Search highlights

When =diff-minimap-search-highlights= is non-nil (the default), every
isearch match appears as a yellow bar in the minimap at the
corresponding row.  The highlights update live on each keystroke, so you
can see where matches are across the file without scrolling.  They
disappear when isearch exits.

To disable:

#+begin_src emacs-lisp
(setq diff-minimap-search-highlights nil)
#+end_src

* How it works

The minimap buffer is written once by =diff-minimap--render=, which:

1. Collects diff data from the configured backend (=diff-hl= overlays or =git diff=).
2. Maps source lines to minimap rows via a scale factor: =total-source-lines / minimap-rows=.
3. Paints each minimap row with an =:extend t= background face (normal background from =fringe=).
4. Applies diff colours as overlays (priority 3) on rows the backend reports as changed — inserted (green), changed (yellow), removed (red).
5. Paints fringe bitmaps in the source buffer on every changed line (when =diff-minimap-fringe-indicators= is non-nil).
6. Stores =source-line= and =source-buf= text properties on each row for click-to-jump.

The viewport indicator and cursor marker are then drawn as Emacs overlays on top of the minimap buffer — they are updated on every =post-command-hook= by =diff-minimap--update-viewport= (no buffer text is touched).

The mode is global: the minimap window lives on one side of the frame and its content follows the selected buffer.  Switching to a file with an available backend triggers a full render; switching to Dired or a buffer without changes clears the minimap until a file buffer is selected again.

A full re-render (=diff-minimap--render=) is triggered only when:

- The buffer content changes (=buffer-chars-modified-tick=).
- The source window is resized (=diff-minimap--last-nrows= changes).
- The active theme changes (=custom-enabled-themes= changes).

Face colours are cached in =defvar=s so the nine =face-attribute= calls run once per full render rather than once per minimap row.

* Dependencies

- Emacs 27.1+
- =cl-lib= (built-in)
- =diff-hl= (optional — only needed for the =diff-hl= backend; the
  =vc= backend works standalone.  When present, =diff-hl= continues to
  provide fringe diff indicators independently of the minimap.)

* Known limitations

- The minimap uses =display-buffer-in-side-window= directly rather than =display-buffer-alist=, so custom =display-buffer-alist= rules may conflict.
- The minimap window has =window-size-fixed= set to ='width= to prevent horizontal resizing on frame split.  Manual width adjustment with =C-x {= / =C-x }= is disabled; change the width via =diff-minimap-width= and toggle off/on.
- The =stipple= viewport style requires an X11 frame; it may not render on Cairo or PGTK backends.

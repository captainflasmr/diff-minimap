A VSCode-style minimap for Emacs (version 0.3.0) that colour-codes every line
by its diff-hl status (added, changed, removed) while also marking the visible
viewport and current cursor row.  Entirely self-contained — no dependency on
the =demap= minimap package.

* Installation

Drop =diff-minimap.el= onto =load-path= and require it:

#+begin_src emacs-lisp
(add-to-list 'load-path "~/.emacs.d/offline-packages/local-packages/diff-minimap")
(require 'diff-minimap)
#+end_src

Or with =use-package= (no =:ensure= since it is a local package):

#+begin_src emacs-lisp
(use-package diff-minimap
  :load-path "~/.emacs.d/offline-packages/local-packages/diff-minimap")
#+end_src

* Usage

~M-x diff-minimap-toggle~ or ~C-x v m~ opens the minimap in a side window.  The
same command closes it.

The minor mode =diff-minimap-mode= is automatically activated when the minimap
opens and deactivated when it closes.  It is a global mode — once toggled on,
the minimap follows the selected buffer automatically, re-rendering for each
file that has =diff-hl-mode= enabled, and clearing when entering Dired or
other non-diff buffers.

Click any row in the minimap to jump the source buffer to that line.

* Viewport styles

The viewport indicator can be selected via =diff-minimap-viewport-style=.
Available styles:

| Style      | Description                                                      |
|------------+------------------------------------------------------------------|
| =edge-bar= | Solid bars at the top and bottom rows of the viewport (default). |
| =filled=   | Solid background across the entire viewport (obscures diffs).    |
| =outline=  | Box outline around the viewport in the cursor colour.            |
| =stipple=  | Bitmap pattern fill + overline/underline edges (diffs visible). |

Set it in your config:

#+begin_src emacs-lisp
(setq diff-minimap-viewport-style 'outline)
#+end_src

* Customisation

** User options

| Variable                        | Default        | Description                                                  |
|---------------------------------+----------------+--------------------------------------------------------------|
| =diff-minimap-font-scale=       | 0.4       | Fraction of default font height for the minimap (1.0 normal) |
| =diff-minimap-width=            | 8         | Width in characters of the side window (at the small font)   |
| =diff-minimap-side=             | 'right    | Side to display the minimap (=left= or =right=)              |
| =diff-minimap-viewport-style=   | 'edge-bar | How to render the viewport indicator (see above)             |
| =diff-minimap-stipple-pattern=  | 'dots-sparse | Bitmap pattern for the =stipple= viewport style          |
| =diff-minimap-colour-source=    | 'diff-hl  | Prefer =diff-hl= fringe faces or built-in =diff= faces      |

See the [[#stipple-patterns][stipple pattern table]] below for predefined values.

** Stipple patterns

The =stipple= viewport style uses an XBM bitmap as a textured background over
the viewport.  Predefined patterns are available via
=diff-minimap-stipple-pattern=:

| Value        | Description                    | Coverage |
|--------------+--------------------------------+----------|
| =dots-sparse=  | Diagonal single-pixel dots     | ~12 %    |
| =diagonal=     | Thin diagonal lines            | ~25 %    |
| =crosshatch=   | Alternating pixel cross-hatch  | ~50 %    |
| =checkerboard= | Checkerboard                   | ~25 %    |
| =dots-dense=   | Denser dot pattern             | ~25 %    |

To use a custom stipple, set =diff-minimap-stipple-pattern= to a list of the
form =(WIDTH HEIGHT DATA)= where DATA is an XBM bitmap string (bytes arranged
left-to-right, top-to-bottom):

#+begin_src emacs-lisp
(setq diff-minimap-stipple-pattern
      '(8 8 . ,(apply #'string '(#x80 #x40 #x20 #x10 #x08 #x04 #x02 #x01))))
#+end_src

** Colour source

The =diff-minimap-colour-source= option controls which Emacs faces are
queried for the diff highlight colours:

- =diff-hl= (default) queries =diff-hl-insert=, =diff-hl-change=,
  =diff-hl-delete= first, falling back to =diff-added=, =diff-changed=,
  =diff-removed=.
- =diff= reverses the priority: built-in diff faces first, diff-hl faces as
  fallback.

Both paths end with the same hardcoded hex fallbacks (=#006600=, =#666600=,
=#660000=) so the minimap always has colour regardless of theme.

Other derived colours (=norm-bg=, =view-bg=, =cur-bg=) always come from
=fringe=, =region=, and =cursor= respectively.

* How it works

The minimap buffer is written once by =diff-minimap--render=, which:

1. Scans every =diff-hl-hunk= overlay in the source buffer.
2. Maps source lines to minimap rows via a scale factor:
   =total-source-lines / minimap-rows=.
3. Paints each minimap row with an =:extend t= background face derived from
   the hunk type (insert = green, change = yellow, delete = red).
4. Stores =source-line= and =source-buf= text properties on each row for
   click-to-jump.

The viewport indicator and cursor marker are then drawn as Emacs overlays on
top of the minimap buffer — they are updated on every =post-command-hook= by
=diff-minimap--update-viewport= (no buffer text is touched).

The mode is global: the minimap window lives on one side of the frame and its
content follows the selected buffer.  Switching to a file with =diff-hl-mode=
triggers a full render; switching to Dired or a buffer without =diff-hl-mode=
clears the minimap until a file buffer is selected again.

A full re-render (=diff-minimap--render=) is triggered only when:

- The buffer content changes (=buffer-chars-modified-tick=).
- The source window is resized (=diff-minimap--last-nrows= changes).
- The active theme changes (=custom-enabled-themes= changes).

Face colours are cached in =defvar=s so the nine =face-attribute= calls run
once per full render rather than once per minimap row.

* Dependencies

- Emacs 27.1+
- =diff-hl= (required at runtime; the minimap is a no-op when
  =diff-hl-mode= is not active)
- =cl-lib= (built-in)

* Known limitations

- The minimap uses =display-buffer-in-side-window= directly rather than
  =display-buffer-alist=, so custom =display-buffer-alist= rules may
  conflict.
- The minimap window has =window-size-fixed= set to ='width= to prevent
  horizontal resizing on frame split.  Manual width adjustment with
  =C-x {= / =C-x }= is disabled; change the width via
  =diff-minimap-width= and toggle off/on.
- The =stipple= viewport style requires an X11 frame; it may not render on
  Cairo or PGTK backends.
